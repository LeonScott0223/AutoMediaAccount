# 后台管理系统开发规划

## C端功能扫描结论

- 当前项目为静态前台，页面包括：首页 `index.html`、商品详情 `detail.html`、订单列表 `orders.html`、订单详情 `order-detail.html`、登录注册 `auth.html`、个人中心 `profile.html`。
- 当前没有真实后端 API，也没有 Model/Entity 代码；商品、公告、支付方式、演示订单、演示用户来自 `web/assets/js/data.js`。
- 当前本地存储包括：订单 `auto-account-orders`、当前用户 `auto-account-user`、用户列表 `auto-account-users`、待支付订单 `auto-account-pending-payment`、公告当天免提醒 `auto-account-notice-hide-date`。
- C端反推后台核心模块：商品管理、卡密库存管理、订单管理、用户管理、网站公告管理、支付方式管理、平台分类管理、系统配置管理、后台管理员管理。

## 一级菜单：商品管理

### 二级菜单：商品列表

搜索字段：

- 商品标题（输入框）
- 账号分类（下拉选择：某音、某手、某视频号、某红薯、某百家号）
- 受众偏好（下拉选择：女粉优势、男粉优势、实名账号、高收藏）
- 实名状态（下拉选择：已实名、未实名、可自定义实名、固定实名）
- 商品状态（下拉选择：上架、下架）
- 价格区间（输入框：最低价格、最高价格）
- 粉丝数区间（输入框：最低粉丝数、最高粉丝数）
- 收藏数区间（输入框：最低收藏数、最高收藏数）
- 创建时间（日期范围）

列表字段：

- 商品ID、商品编号、商品标题、账号分类、受众偏好、价格、库存数量、粉丝数、收藏数、男女粉比例、实名状态、商品状态、排序值、创建时间

列表操作：

- 查看详情、编辑、删除、上架/下架、复制商品、管理卡密

新增/编辑表单字段：

- 商品编号（必填，文本，唯一，长度 2-50）
- 商品标题（必填，文本，长度 2-120）
- 商品副标题（必填，文本，长度 2-255）
- 账号分类ID（必填，下拉选择，必须存在于账号分类表）
- 受众偏好ID（必填，下拉选择，必须存在于受众偏好表）
- 商品价格（必填，金额，必须大于等于 0）
- 商品图片（必填，图片地址，长度 1-255）
- 粉丝展示值（必填，文本，例如 2.1w，长度 1-20）
- 粉丝数值（必填，整数，必须大于等于 0）
- 收藏展示值（必填，文本，例如 8.7w，长度 1-20）
- 收藏数值（必填，整数，必须大于等于 0）
- 性别标签（必填，文本，例如 女粉 76%，长度 1-30）
- 男女粉比例（必填，文本，例如 女粉 76% / 男粉 24%，长度 1-80）
- 粉丝画像（必填，多行文本，长度 1-500）
- 是否已实名（必填，单选：是、否）
- 实名展示文案（必填，文本，长度 1-20）
- 实名模式（必填，下拉选择：固定实名、可自定义实名）
- 商品介绍（必填，富文本/多行文本，长度 1-2000）
- 使用说明（必填，富文本/多行文本，长度 1-2000）
- 常见说明（必填，多行文本，每行一条，最多 20 条）
- 排序值（选填，整数，默认 0）
- 商品状态（必填，单选：上架、下架）
